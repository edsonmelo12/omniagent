# Anti-Repetition - Amiclube

## Purpose

Este arquivo define como detectar e impedir que as artes da Amiclube repitam a mesma lógica visual.

O problema que queremos evitar não é apenas “parecido”. É o lote inteiro parecer a mesma estrutura com texto trocado.

## What counts as repetition

Considere repetição quando duas peças compartilham:

- mesmo ponto focal
- mesma hierarquia de texto
- mesma densidade
- mesmo tratamento de fundo
- mesma posição do CTA
- mesma lógica de composição
- mesma temperatura visual

## Mandatory alternation rules

### 1. Alternate focal logic

Se uma peça usa headline dominante, a próxima deve preferir outra lógica:

- prova dominante
- bastidor dominante
- comparação dominante
- fechamento dominante

### 2. Alternate text weight

Não deixar todo o lote depender de:

- título grande
- subtítulo pequeno
- CTA padronizado

Alterne entre:

- peça com título dominante
- peça com prova dominante
- peça com fluxo modular
- peça com síntese forte

### 3. Alternate density

Se uma peça é cheia, a próxima deve respirar mais.

Se uma peça é mais limpa, a próxima pode trazer mais matéria visual.

### 4. Alternate balance

Evite usar o mesmo tipo de composição em sequência:

- centralizada
- assimétrica
- em coluna
- em blocos
- em camadas

## Similarity warning matrix

Use este raciocínio:

- 1 coincidência: normal
- 2 coincidências: atenção
- 3 ou mais coincidências: reprovar ou redesenhar

Coincidências avaliadas:

- centro visual
- distribuição de texto
- cor dominante
- nível de textura
- presença de moldura
- tipo de CTA
- peso do fundo

## Batch-level rules

### In the same batch

O lote precisa de uma sequência legível, mas não repetitiva.

Exemplo de boa sequência:

1. manifesto
2. prova
3. bastidor
4. fechamento

Exemplo de sequência ruim:

1. editorial
2. editorial
3. editorial
4. editorial

### Across batches

O batch seguinte deve evoluir.

Permita repetir apenas se:

- houver motivo estratégico
- o formato exigir continuidade
- a variação interna já for suficiente

## Visual anti-patterns

### Never repeat

- mesmo layout-base com texto trocado
- mesma composição central em todas as peças
- mesmo uso de fundo escuro em sequência
- mesmo CTA em mesma posição
- mesmo ritmo de blocos
- mesmo foco em todas as peças

### Always do

- comparar com o batch anterior
- alternar foco e hierarquia
- buscar diferença real de leitura
- preferir diversidade com coerência
- revisar se a peça acrescenta variedade ao conjunto

## Detection prompts for the skill

Pergunte internamente:

- se eu escondesse o texto, ainda pareceria a mesma peça?
- se eu trocasse o título, a composição ainda estaria igual?
- essa direção já apareceu no lote anterior?
- existe um motivo real para essa estrutura?
- o que está mudando de verdade aqui?

## Approval rule

Se a peça parece “bonita mas igual”, reprovar.

Se ela é clara, útil e diferente na leitura, aprovar.

## Final rule

A skill não deve tentar apenas salvar a estética.
Ela deve proteger o conjunto contra monotonia.
