# Write SEO + GEO Content

Use this skill when drafting or refining a long-form page that must perform in search and AI-driven discovery.

## Hard Rule

Never write the final article before completing pre-writing research and evidence locking.

## Pre-Writing Checklist

- topic and page goal
- search intent
- audience and funnel stage
- thesis and information gain
- proof sources and freshness
- entity references
- CTA direction

## Drafting Rules

1. Preserve the chosen thesis and structure family.
2. Add a direct answer path near the top.
3. Keep the article scannable on mobile.
4. Use proof where claims matter.
5. Make the article reusable for repurposing without sounding generic.

## Opening Summary Rule

- A top summary block is optional, never mandatory.
- Use `TL;DR`, `Resumo executivo`, or another short-answer opening only when it improves search intent fit, retrieval, or reader utility.
- Do not repeat the same opening pattern across every article in a cluster or client account.
- The chosen opening pattern must be justified by the brief or architecture, not by habit.
- If the article works better with a direct hook, scene, question, or contrarian opening, keep that instead of forcing a summary block.

## Evidence Discipline

- Label claims that need proof.
- Distinguish confirmed proof from hypothesis.
- Do not invent source support.
- Prefer current, attributable, and defensible references.

## Required Final Blocks

- title tag
- meta description
- structured body
- FAQ
- source notes
- schema recommendation
- CTA

## Conditional Final Blocks

- optional top summary block (`TL;DR`, `Resumo executivo`, direct-answer block, or equivalent)
- only include when it materially improves extractability, scanability, or intent match
- if omitted, the intro still needs a clear answer path near the top

## Quality Gate

Reject and redo if:

1. The article could rank only by being longer, not better.
2. The structure collapses back into a generic mold.
3. The GEO pass adds extractability but removes editorial personality.
4. A `TL;DR` or similar top block was added by default rather than by editorial need.
