# Creative Validation Checklist

Client: active client
Cycle: current cycle
Purpose: Validate that the visual direction is varied, readable and production-ready before render and approval.

## How to Use

Run this checklist after the visual direction is drafted and before the final render manifest is approved.

- Mark each item as pass, conditional pass or fail.
- If any item fails, the visual direction needs revision before rendering.
- If the batch feels too similar to the previous one, apply the anti-repetition layer again before approval.

## Validation Gates

### 1. Clarity and value
- Does the piece communicate one clear idea?
- Is the value obvious in under a few seconds?
- Is the hook legible on mobile without zooming?
- Does the layout avoid unnecessary clutter?

Pass criteria:
- one dominant idea
- visible value signal
- no competing headline hierarchy

### 2. Educational structure
- Does the piece teach, explain or organize a useful idea?
- Is the progression easy to follow?
- Are the steps or blocks visually distinct?
- Is the density appropriate for the format?

Pass criteria:
- logical sequence
- clear scanning path
- no oversized text blocks

### 3. Proof and trust
- Is there a visible proof element?
- Does the proof feel tied to the message instead of decorative?
- Is the evidence easy to identify on the first pass?
- Does the piece avoid vague credibility claims?

Pass criteria:
- proof is visible
- proof is specific
- proof supports the thesis

### 4. Closing strength
- Does the piece feel like a conclusion or a thesis?
- Is the final message memorable?
- Does the close match the batch objective?
- Does the ending feel intentional?

Pass criteria:
- clear closing statement
- strong final frame
- no loose visual logic

### 5. Anti-repetition
- Does this piece use a different focal logic from the previous one?
- Does it change hierarchy, not just color or copy?
- Does it avoid reusing the same template behavior?
- Does it improve variation across the batch?

Pass criteria:
- different structure family
- different focal point
- different reading rhythm

### 6. Source sanity
- Is the source class the least risky class that fits the piece?
- If external imagery is used, is the source search note recorded?
- Does the source search note include candidate types and selection rationale?
- Is the asset still stronger than a text-first or typographic alternative?

Pass criteria:
- source class is explicit
- source search note is documented
- source rationale is traceable

## Batch-Level Rule

If 3 or more pieces in the same batch share the same composition logic, the batch must be revised before render.
