## Production Notes

- Build the Instagram asset first for every day.
- Render and record the master asset before downstream variants.
- Derive Facebook only when the piece benefits from redistribution.
- Derive Pinterest only when the piece is visually save-worthy.
- Use Stories as light support, not as a separate creative system.
- Use the product image only when it increases proof, trust or value.
- Keep all layouts readable on mobile before styling details.
- Use CC0/Public Domain first when a sourced visual is needed.
- Keep the source search note attached to any external asset handoff.

## Execution Order

1. Build IG master.
2. Render and record the master asset.
3. Produce proof/product asset if required.
4. Adapt for FB and PIN.
5. Validate mobile readability.
6. Publish only after review.
