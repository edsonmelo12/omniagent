# Creative Execution Plan

Client: active client
Cycle: current cycle
Mechanism: template/render production, no AI image model for the first pass

## Global Guardrails

- Validate every batch with the [Creative Validation Checklist](validation-checklist.md) before render.
- Do not approve a day if it reuses the same composition logic as the previous one.
- Use the checklist to confirm clarity, proof, closing strength and anti-repetition.
- Choose the source class before any image-led day.
- Record the source search note before render when external imagery is required.

## Legend

- `editorial card`: text-first graphic
- `product shot`: direct view of the product or offer
- `process shot`: hands, making, finishing, work in progress
- `comparison card`: before/after or weak/strong presentation
- `proof card`: benefit, result, support or reassurance
- `recap card`: weekly summary or closing thesis

## Global Reuse Rules

- Use the original asset as the base.
- Rebuild the layout only when the channel needs it.
- Keep the product visible when the post depends on trust or value.
- Avoid product images on pure clarity or manifesto posts.
- Never ship a mobile-unsafe layout.
- Never skip source documentation when using external imagery.

## Production Order

1. Build the Instagram master asset.
2. Render and log the master asset.
3. Derive the Facebook adaptation when needed.
4. Derive the Pinterest version when the piece is save-worthy.
5. Run a mobile readability review.
6. Approve for scheduling only after the visual check passes.
