# Source Selection Guide

Client: active client
Cycle: current cycle
Purpose: choose the safest and most useful source class for each piece before production.
Niche rule: prioritize images that include direct signals from the client's niche table before choosing neutral lifestyle imagery.
Blog rule: when the piece is a blog post, treat the article thesis as the primary filter and only source an image if it improves clarity, proof, scanability or editorial authority.

## Decision Flow

1. Decide whether the piece needs an external source at all.
2. If yes, choose the source class.
3. Prefer the least risky license that still fits the visual need.
4. Record the source class, source search note and rationale before render.

## Source Classes

### A. No external source
Use when the piece is mostly typography-led or graphically constructed.

Best for:

- editorial cards
- manifesto posts
- thesis slides
- simple checklist cards

### B. CC0 / Public Domain support image
Use when maximum freedom and reuse are important.

Best for:

- backgrounds
- simple context images
- generic objects
- texture references

### C. Unsplash / Pexels support image
Use when a high-quality free photo is needed and the commercial license is acceptable.

Best for:

- lifestyle scenes
- human hands / workspace mood
- environmental support shots
- visually rich but generic context

### D. Wikimedia Commons reference
Use only when the exact file license is verified and the content is specifically useful.

Best for:

- niche educational context
- historical or factual material
- special references that stock sites do not cover

### E. Pixabay fallback
Use when a free stock-style image is enough and the license summary fits the use.

Best for:

- generic stock visuals
- simple supporting imagery
- low-risk placeholder support

## Practical Rules

- If the design is strong without a photo, do not force one.
- If the piece is proof-led, prioritize an image that supports evidence.
- If the piece is backstage-led, prioritize a real-world context shot.
- If the piece is educational, prioritize clarity over visual complexity.
- If the piece is manifesto-led, keep sourcing minimal.
- Prefer `nicho_direto` first, then `nicho_adjacent`, then `nicho_metaforico`, then `fallback_neutro`.
- If the image does not say anything about the niche, keep looking.
- If the client niche is specific, reject images that visually identify a neighboring craft or category, even if the asset is otherwise beautiful.
- For blog posts, prefer a custom editorial cover or diagram over generic stock photography unless the article truly depends on context imagery.
- Use only free/public sources by default; paid stock does not belong in the standard sourcing path.
- If a paid source is ever proposed, it must be explicitly approved and documented outside the default flow.

## Metadata to Record

For every sourced asset, save:

- source class
- source name
- URL
- license type
- attribution requirement
- intended use
- source search note
- selection rationale

## Final Rule

Source material should make the piece stronger.
If it only fills space, remove it.
If possible, the source should read as part of the client's world instead of a generic stock image.
Default to free or public sources only. Paid sources are an exception, not a workflow default.
