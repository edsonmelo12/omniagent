# Daily Validation Template

Client: active client
Cycle: current cycle
Purpose: Reuse the validation checklist for each day of the creative cycle before render and approval.

## Usage

Fill this template once per day before the render step.

- If the answer is fail, do not export the asset.
- If two or more answers are conditional pass, revise the direction before export.
- If the piece feels too similar to the previous day, apply the anti-repetition layer again.

## Daily Gates

### Clarity and value
- One clear idea
- Hook readable on mobile
- Value visible quickly

### Educational structure
- Sequence is easy to follow
- Blocks are distinct
- Density fits the format

### Proof and trust
- Proof is visible
- Proof is specific
- Proof supports the thesis

### Closing strength
- Ending feels intentional
- Final frame is memorable
- Close matches the day objective

### Anti-repetition
- Focal logic differs from the previous day
- Hierarchy changes, not just copy or color
- Composition logic brings new rhythm to the batch

### Source sanity
- Source class is the least risky class that fits the piece
- No external source is used when the piece is type-led and can stand alone
- Any sourced asset has a documented source search note, license and URL

## Daily Decision Rule

Pass all gates before render.

If the day is only visually different but not structurally different, revise it.
