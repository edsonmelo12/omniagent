# Social Growth Creative Playbook

Client: active client
Cycle: current cycle
Purpose: Consolidate the operational rules for planning, validating, rendering, reviewing and scheduling creative assets.

## Core Principle

The creative system is not only about making assets.
It is about making distinct assets that still feel like one campaign.

For the active client, this means:

- use the dedicated creative-director skill
- apply anti-repetition rules every day
- validate each day before render
- source support imagery through the free/public source policy
- choose the source class through the source-selection guide
- record a source search note, candidate types and rationale whenever external imagery is used
- for blog posts, treat imagery as optional unless it improves clarity, proof, scanability or editorial authority
- keep the review and approval gates explicit

## Operating Sequence

1. Define the day objective.
2. Choose the primary asset type.
3. Assign the direction family.
4. Run the daily validation template.
5. Render only after validation passes.
6. Review the output against the discovery-optimized blog, repurpose package and visual direction.
7. Approve the schedule only after the checklist passes.

## Validation Rule

Do not render or schedule a piece if:

- the hook is unclear on mobile
- the visual logic repeats the previous asset
- the proof is implied but not visible
- the close does not feel intentional
- the batch loses variation
- the source search note is missing for an external-image blog asset

## Daily Production Rule

The day is only ready when:

- the direction family is named
- the validation checklist passes
- the render manifest is complete
- the schedule plan includes the approved asset

## End-of-Cycle Rule

The cycle should end with:

- a clear thesis
- a visible proof trail
- a recurring but non-repetitive visual system
- assets that feel related without feeling duplicated
- source metadata and rationale preserved for every external image
