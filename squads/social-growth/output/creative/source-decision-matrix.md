# Source Decision Matrix

Client: active client
Cycle: current cycle
Purpose: define the default free-source class by post type so the system can decide faster and with less risk.
Niche rule: when external imagery is used, prefer assets that show the signals from the client's niche table before falling back to neutral imagery.
Blog rule: when the asset is for a blog post, prefer a suggestive visual by default and use imagery only when it improves clarity, proof, scanability or editorial authority.
Blog sourcing rule: when external imagery is needed for a blog asset, document a free/public source search note with candidate types, license notes and the selection rationale before production.
The handoff should always include source class, source search note and selection rationale.

## Decision Matrix

| Post Type | Default Source Class | Use External Source? | Notes |
|---|---|---|---|
| Editorial card | No external source | No | Let typography carry the piece.
| Checklist card | No external source | No | Keep the layout graphic-led and clean.
| Comparison card | No external source | No | Use internal composition instead of source imagery.
| Recap card | No external source | No | Focus on summary and thesis.
| Manifesto carousel | No external source | No | Keep the ending strong and typographic.
| Educational carousel | No external source | Usually no | Use source only if a real context image improves clarity.
| Product shot + proof card | Unsplash / Pexels or CC0 / Public Domain | Yes | Use when a real-world support image strengthens value.
| Process shot / reel | Unsplash / Pexels or CC0 / Public Domain | Yes | Use when hands, materials or workspace context matter.
| Proof reel | Unsplash / Pexels or CC0 / Public Domain | Yes | Use to support evidence and trust.
| Reassurance carousel | No external source or CC0 / Public Domain | Maybe | Use only if a calm context image improves trust.
| Value card | Unsplash / Pexels or CC0 / Public Domain | Yes | Use when proof depends on visible context.
| Text-first post | No external source | No | Avoid forcing imagery where it weakens the message.
| Blog editorial post | No external source | Usually no | Use a custom editorial cover or a niche-relevant support image as the default visual.
| Blog proof article | Unsplash / Pexels or CC0 / Public Domain | Yes | Use when the article depends on visible proof, context or process.
| Blog educational article | No external source or CC0 / Public Domain | Usually no | Use diagrams, frameworks or typographic covers before generic photography.

## Practical Rules

- If the piece is type-led, do not add source imagery just to fill space.
- If the piece depends on proof, source an image that reinforces the claim.
- If the piece depends on backstage or craft, source real context whenever possible.
- If the piece is a manifesto or thesis, keep source usage minimal.
- If a sourced image is needed, prefer `nicho_direto` or `nicho_adjacent` over neutral stock.
- Never upgrade an adjacent craft into the wrong craft: a visually specific neighboring niche is a rejection, not a compromise.
- For blog posts, do not use a generic image just to satisfy a template. The article must justify the asset.
- For blog posts, include the source search note and rationale in the visual direction before render starts.

## Allowed Source Classes

- No external source
- CC0 / Public Domain
- Unsplash / Pexels
- Wikimedia Commons
- Pixabay

## Final Rule

The default should be the simplest source class that still makes the piece better.
If an image is used, it should carry at least one signal from the niche instead of reading as generic filler.
