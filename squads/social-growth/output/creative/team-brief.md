# Team Brief

Client: active client
Cycle: current cycle

## Goal

Run a daily organic campaign that builds authority, proof and conversion readiness for artisans.

## Core Message

The client helps its audience with clarity, proof and routine.

## Strategy

- Primary: Autoridade para gerar prova
- Secondary: Conversão para demonstrar valor
- Safety: Plano B para reduzir risco

## Channels

- Instagram: primary daily feed
- Stories: daily support
- Facebook: redistribution support
- Pinterest: evergreen discovery support
- TikTok: inactive until live confirmation

## Production Method

- Use template/render production.
- Do not use an AI image model on the first pass.
- Use the product image only when it increases proof, trust or value.
- Choose the source class with the source-decision matrix before using any external image.
- Record the source search note and rationale before external-image production.
- Keep layouts mobile-readable before stylistic refinement.
- Validate every client batch with [Creative Validation Checklist](validation-checklist.md) before render approval.

## Responsible Roles

- `creator`: copy and platform variations
- `visual-director`: visual direction and creative briefs
- `creative-renderer`: template execution, export and asset manifest
- `reviewer`: quality control of copy and layout
- `scheduler`: calendar and delivery order

## What to Produce

1. Instagram master asset
2. Render manifest and final asset exports
3. Facebook adaptation when useful
4. Pinterest adaptation when useful
5. Stories support frames
6. Reel cover and proof treatment

## Visual Rules

- Editorial, practical and confident
- High contrast covers
- 2 fonts max
- 4-color palette max
- One dominant idea per slide or frame
- Proof must be visible, not implied

## Product Usage Rules

- Use product shots on proof and value posts.
- Use process shots on bastidor and evidence posts.
- Use editorial cards on clarity and thesis posts.
- Use comparison cards on objection and before/after posts.
- Use recap cards on summary posts.
- Use the free-source policy, source selection guide and source search note when an asset needs external imagery.

## Channel Reuse Rules

- Instagram is the master version.
- Facebook should be shorter and denser only when needed.
- Pinterest should be vertical, title-led and evergreen.
- Stories should reinforce hooks, proof and interaction.

## Execution Order

1. Build the Instagram master asset.
2. Run the creative validation checklist.
3. Render the final asset and record the manifest.
4. Derive proof/product assets when needed.
5. Adapt for Facebook and Pinterest when useful.
6. Review mobile readability.
7. Publish after visual check.

## Non-Goals

- No AI image generation in the first pass.
- No separate creative system per channel.
- No product image where it does not strengthen the message.
- No layout that is hard to read on mobile.
