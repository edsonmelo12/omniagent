# Production Package

Client: active client
Cycle: current cycle
Mechanism: template/render production, no AI image model for the first pass

## Production Rules

- Use the approved discovery-optimized blog as the long-form text source when the cycle includes blog content.
- Use the approved repurpose package as the source for native social adaptations.
- Use the approved content production package as the final assembly source.
- Use the approved visual direction as the layout source.
- Validate the direction against the creative checklist before export.
- Select the source class before production when an asset needs external imagery, and preserve the source search note plus rationale from the visual direction.
- Build each piece as a platform-native visual asset.
- Record each final export in a render manifest with path, format, size and status.
- Keep the product visible when it increases proof, value or trust.
- Use text-first layouts when the piece depends on clarity or education.
- Use comparison or before/after when the piece depends on persuasion.
- Keep every asset readable on mobile before anything else.

## Asset Types

- `product shot`: direct view of the product or offer
- `process shot`: hands, making, work in progress, finishing
- `editorial card`: text-first social graphic
- `comparison card`: before/after or weak/strong presentation
- `proof card`: benefit, result, support or reassurance
- `recap card`: weekly summary or thesis close

## Source Classes

- `No external source`: pure typography or graphic-led pieces
- `CC0 / Public Domain`: maximum freedom support images
- `Unsplash / Pexels`: free photography with commercial-friendly licenses
- `Wikimedia Commons`: reference-only when the exact file license is checked
- `Pixabay`: fallback stock-style visuals

## Source Decision Matrix

- editorial card -> No external source
- checklist card -> No external source
- comparison card -> No external source
- recap card -> No external source
- manifesto carousel -> No external source
- educational carousel -> usually No external source
- product shot + proof card -> Unsplash / Pexels or CC0 / Public Domain
- process shot / reel -> Unsplash / Pexels or CC0 / Public Domain
- proof reel -> Unsplash / Pexels or CC0 / Public Domain
- reassurance carousel -> No external source or CC0 / Public Domain
- value card -> Unsplash / Pexels or CC0 / Public Domain
- text-first post -> No external source

## Channel Reuse Rules

### Instagram
- use the native layout for the original piece
- keep the strongest cover on the first screen
- keep proof close to the hook when possible

### Facebook
- reuse the strongest visual from the original piece
- shorten the caption
- reduce layout density

### Pinterest
- prefer the most save-worthy visual
- use vertical layout and title-led framing
- keep the message evergreen

## Production Checklist

- hook visible at a glance
- product image used only when it adds proof
- source class selected when external imagery is needed
- source search note includes candidate types and rationale
- paid stock is not used unless explicitly approved outside the default flow
- no overloaded text blocks
- CTA visible and simple
- mobile readability confirmed
- no AI image generation required for the first pass
- creative validation checklist passed

## Delivery Standard

- one final visual brief per post
- one rendered asset or manifest entry per post
- one free/public source search note per external-image post
- one production note per channel adaptation
- one review pass before scheduling
