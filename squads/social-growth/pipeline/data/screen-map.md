# Screen Map

## Purpose

Define the minimum set of screens for the multi-agency platform.

The screen map is intentionally lean so the product can be built and validated quickly before adding complexity.

## Public Entry

### Login
- sign in
- password reset
- agency identification

### Agency Selector
- switch between agencies
- create new agency if permitted

## Agency Area

### Dashboard
- client count
- active cycles
- proposals in progress
- registros do cliente precisando de atualização
- social health summary

### Clients
- list of clients
- filters by status and segment
- create client
- open client workspace

### Proposals
- proposal list
- proposal status
- latest version
- approval status

### Approvals
- pending approvals
- approved items
- rejected items
- review notes

### Monitoring
- latest social health scores
- KPI snapshots
- actions recommended
- historical trend

## Client Workspace

### Overview
- client identity
- site
- social profiles
- current stage
- last update

### Social Intelligence

### Registro do cliente

- offer priority

- corrections

### Market Research

### Proposal

### Content

### Monitoring

- notes
- action recommendations

## Operations Area

### Source Intake
- website links
- social links
- documents
- discovery notes

### Evidence Library
- source links
- screenshots
- exports
- confidence labels

### Builder Views
- intelligence summary
- record builder
- proposal builder
- schedule builder

## Screen Principles

- Keep the first release compact.
- Do not split one logical step across too many screens.
- Every screen should support an action, not just display data.
- Agency views should summarize.
- Client views should explain and allow correction.

## Recommended Navigation

1. Login
2. Agency dashboard
3. Client workspace
4. Social intelligence
5. Registro do cliente
6. Proposal
7. Content
8. Monitoring

## Final Recommendation

The platform should start with a very small number of screens that map directly to the operating flow.

That keeps the product understandable for the agency and makes the transition from Markdown-based operations much easier.
