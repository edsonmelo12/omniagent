# Blog Structure Families

Use this matrix to avoid producing every post with the same editorial skeleton.
The family must be chosen before drafting and preserved through optimization and review.

## 1. Teardown Critico

- Purpose: break down a weak belief, pattern or practice.
- Best for: B2B credibility, GEO, process critique, positioning problems.
- Arc: problem -> symptom -> mechanism -> proof types -> application -> limits -> conclusion.
- Repetition guardrail: do not reuse the same "problem -> 3 steps -> FAQ -> conclusion" order every time.

## 2. Framework Operacional

- Purpose: turn a topic into a usable system.
- Best for: playbooks, prioritization, checklists, workflow design.
- Arc: why it matters -> components -> sequence -> example -> failure modes -> CTA.
- Repetition guardrail: vary the opening shape and the proof placement.

## 3. Contrarian Guide

- Purpose: challenge the default assumption and replace it with a better model.
- Best for: opinionated thought leadership, category education, strategic reframing.
- Arc: common belief -> why it breaks -> better frame -> practical use -> objection handling -> conclusion.
- Repetition guardrail: keep the counterpoint visible and avoid turning it into a generic list.

## 4. Comparison / Decision

- Purpose: help the reader choose between options or approaches.
- Best for: tool selection, channel selection, format choice, tradeoff analysis.
- Arc: decision context -> option A/B/C -> criteria -> tradeoffs -> recommendation -> FAQ.
- Repetition guardrail: do not force a teardown structure if the topic is actually comparative.

## 5. Case Study / Proof Story

- Purpose: show how an idea worked in a concrete situation.
- Best for: proof-led content, result stories, implementation narratives.
- Arc: context -> constraint -> decision -> execution -> result -> lesson -> CTA.
- Repetition guardrail: keep the case specific; do not turn it into an abstract framework article.

## 6. Playbook / Checklist

- Purpose: give a reader a direct operational sequence.
- Best for: tactical content, launch prep, audits, implementation steps.
- Arc: when to use -> what to prepare -> step sequence -> checklist -> mistakes -> next move.
- Repetition guardrail: avoid over-explaining the basics; keep the sequence decisive.

## 7. Manifesto / Position Piece

- Purpose: declare a strategic point of view and the operating principle behind it.
- Best for: category creation, positioning, brand authority.
- Arc: belief -> tension -> principle -> implications -> evidence -> closing position.
- Repetition guardrail: do not use this family for every topic; it should be reserved for high-authority angles.

## Selection Rule

- The architecture step must choose one family intentionally.
- Neighboring posts should not repeat the same family unless there is a strong editorial reason.
- The writer must preserve the family-specific arc.
- The optimizer may improve clarity and retrieval, but may not flatten the family back into the default blog mold.
- The selected family, opening shape, proof placement, nuance placement and close shape must be written into the blog brief and ledger before drafting.
