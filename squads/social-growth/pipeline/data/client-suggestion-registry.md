# Client Suggestion Registry

## Purpose

Central reference for client-specific suggestion patterns used by this chat as the operational terminal.

## Active Client Patterns

### Active client

- Start from the latest database-backed client record.
- Keep the cycle centered on the strongest educational, proof or conversion angle from the client table.
- Favor guidance on pricing, selling, organization, proof of value or the niche-specific equivalent.
- Use catalog items, offers and proof points as content inputs, not as the whole strategy.
- If a product focus exists, use the product record from the database before the next recommendation.
- If an organic campaign exists, use the latest campaign record or generated review copy as the sequence baseline.
- If the cycle includes long-form content, add blog architecture, blog draft, discovery optimization and content repurpose before the content production package.
- If the offer changes, refresh the client record first.
- If the audience changes, refresh the client record first.
- If the channel priority changes, refresh market research before content.
- If the client asks for a consolidated proposal, keep it as a separate branch after strategy and before content. If not, skip it and stay in the research-to-strategy-to-content flow.

### Real Engenharia

- Start from the latest database-backed client record.
- Treat the portfolio as multiple products, not one generic offer.
- Identify the commercially strongest empreendimento first.
- If the priority project changes, rebuild strategy around that project first.

### Porto Real

- Start from the latest database-backed client record.
- Emphasize practical value, decision clarity and fit for the buyer's moment.
- Keep the message grounded in cost-benefit, function and location.
- If the commercial priority changes, update the client record before the next cycle.

## Fallback Rule

If no client-specific pattern exists yet:

1. client record
2. market research
3. social intelligence
4. strategy
5. proposal, only when explicitly requested or checkpointed
6. content plan
7. blog architecture, when long-form content is in scope
8. blog draft
9. discovery optimization
10. content repurpose
11. content production package
12. schedule
13. monitoring

## Working Rule

The registry is intentionally small and opinionated.
Add a new client pattern only when the client has enough artifacts to support a stable recommendation.
