# Contextual Market Intelligence

## Purpose

Analyze the client's relevant market by context, not by a fixed geographic rule.

The market frame may be:
- local;
- regional;
- national;
- niche;
- hybrid.

The module exists so the squad can read the real competitive frame before it writes a proposal, a benchmark or a content brief.
When the client discovers buyers through search or AI systems, this module also records the site's GEO / AI discoverability posture at a high level.

## Scope Resolution

1. Prefer explicit context from the client, product record or notes.
2. If context is missing, infer it from the website, offer structure, service pages, social footprint and location cues.
3. If the scope is still unclear, default to country-level coverage and keep the confidence visible.
4. If the client has multiple products or brands, resolve scope per product or per brand when needed.

## What It Must Collect

### Scope
- scope type
- scope value
- scope basis
- objective of the analysis
- primary audience
- product or service frame
- presence digital map basis: site, blog and social channels as observed public signals

### Market Frame
- direct competitors
- aspirational references
- substitute offers
- geographic relevance when applicable
- niche relevance when geography is not the main axis

### Numeric Presence
- channels found
- channels active
- followers or subscribers
- posts or publications
- last visible post age
- visible reputation count and rating when available

### AI Discoverability
- entity clarity
- service clarity
- answerability
- proof density
- sourceability
- structured data readiness
- retrieval friendliness
- GEO confidence score

### Derived Scores
- presence score
- consistency score
- proof score
- conversion score
- maturity score
- GEO / AI discoverability score
- confidence score

### Data Status
- confirmed
- observed
- inferred
- not confirmed

## Presentation Rule

The executive output must use numbers or numeric states.

- If a value is known, show the number.
- If a value is not known, mark it as `not confirmed`.
- If a value is inferred, label it as inferred.
- Do not present a strong commercial claim without at least one numeric signal or a confirmed source.

## Reuse Rule

This output feeds:
- social intelligence;
- client record;
- market research;
- proposal building;
- monitoring;
- onboarding.

## Decision Rule

The system should avoid asking for missing context unless the missing input changes the scope materially.
If the system can infer the scope from public evidence, it should do so and keep the confidence visible.

## Audience Rule

Audience and persona should be treated as hypotheses derived from evidence:

- primary audience
- secondary audience
- decision-maker
- influencer
- buyer context

If evidence is weak, mark the gap instead of inventing a fictional persona.
