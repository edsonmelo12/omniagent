# Account Launch Kit

## Purpose

Use this kit whenever a new client enters the `social-growth` squad. The goal is to turn links, documents and notes into a usable operating record as fast as possible.

## Inputs

- website
- Instagram
- Facebook
- TikTok
- Pinterest
- product catalog
- brochures
- proposals
- PDFs
- docs
- notes from the client

## Step 1 - Capture Sources

- collect all public links and uploaded files;
- persist them in the intake record and mark unclear or weak sources as pending validation;
- keep the intake editable after onboarding.

## Step 2 - Build the Client Record

- extract company name, offer, audience and tone;
- identify main products and services;
- identify the highest-value segment;
- record proof points and market notes;
- write everything into the database-backed client record.

## Step 3 - Add Market Context

- summarize the category and the main customer pain points;
- identify 3 to 5 opportunities;
- record the strongest channels and content habits;
- note any assumptions or validation risks;
- save the result in the database-backed market context record.

## Step 4 - Define the Weekly System

- set one core theme for the week;
- choose the primary platform for trust;
- choose the secondary platform for discovery;
- assign one CTA for the cycle;
- confirm a realistic cadence.

## Step 5 - Review and Schedule

- build the first blog architecture when the launch includes long-form content;
- create the first blog draft from the approved architecture;
- optimize the blog for discovery before social reuse;
- repurpose the approved blog into native social assets when multichannel output is needed;
- create the first content production package only after the blog, discovery and repurpose outputs are approved;
- review for clarity, fit and CTA strength;
- save approval before scheduling;
- build a simple publication order;
- keep the plan executable for a small team.

## Maintenance Rules

- If the offer changes, refresh the client record first.
- If the audience changes, refresh the client record first.
- If a source is wrong, correct the record and preserve the note.
- If a new channel becomes relevant, add it to the record and the strategy.
