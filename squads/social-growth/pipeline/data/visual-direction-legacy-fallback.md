# Visual Direction Legacy Fallback

Use this only when the JSON manifest path is blocked by an explicit CEO approval path.

## Keep

- one selected style
- one visual system
- one brief per asset
- explicit blog featured-image direction when a blog asset exists
- free/public source search notes only when image research is active
- one `Skill Invocation Ledger`

## Avoid

- replacing the JSON path as the default
- inventing a second visual logic
- skipping the production gate or client DNA gate
- manual HTML generation in the visual direction step

## Output Rule

Keep the markdown brief and only use it for exceptional styles or assets outside Design System coverage.
