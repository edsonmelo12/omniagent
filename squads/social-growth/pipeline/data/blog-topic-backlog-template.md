# Blog Topic Backlog Template

## Purpose

Create a scored, evidence-aware topic backlog before selecting the next blog brief.

This template forces topic quality and prevents random topic selection.

## Output Structure

```
# Blog Topic Backlog

## Backlog Scope
- cycle id:
- generated at:
- owner:
- topics generated: [10-20]

## Ranked Topics
| Rank | Topic | Primary Keyword | Intent | Funnel Stage | Family Candidate | Proof Level | Trend Source | Signal Date | Validity Window | Confidence | Impact (0-5) | Feasibility (0-5) | Proof Readiness (0-5) | Total |
|---|---|---|---|---|---|---|---|---|---|---|---:|---:|---:|---:|

## Ready Now (Top 3)
1. [topic] - [why now]
2. [topic] - [why now]
3. [topic] - [why now]

## Rejected Topics
| Topic | Reason Code | Notes |
|---|---|---|

## Reason Codes
- `R1`: weak proof
- `R2`: weak or stale trend signal
- `R3`: duplicate angle
- `R4`: low business relevance
- `R5`: execution too expensive for current cycle
```

## Usage Rule

The blog brief step must select one topic from the `Ready Now (Top 3)` list unless a documented override is provided.
